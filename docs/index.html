3 完整操作案例導覽

📋 案例說明

本案例示範 ACME Corp 訂購 SKU-001 電子零件 A型 3,000 個 的完整 OTD 流程。每一步都有 API 指令和預期回傳結果。

客戶: ACME Corp 料號: SKU-001 數量: 3,000 PC 目標交期: 2026-06-01
📋
Step 1 — 查詢料號與客戶資料
確認料號存在、價格、產能；確認客戶資料和付款條件。
GET /api/v1/items/SKU-001
GET /api/v1/customers/CUST-001
{"item_code": "SKU-001", "description": "電子零件 A型", "lead_time_days": 7, "daily_capacity": 2000}
🔍
Step 2 — ATP/CTP 交期試算
ATP 確認可承諾數量，CTP 確認產能能否在交期前完成。
POST /api/v1/atp/check
{ "item_code": "SKU-001", "quantity": 3000, "request_date": "2026-05-20" }
{"available": 3500, "promise_date": "2026-05-27", "can_promise": true}
POST /api/v1/ctp/check
{ "item_code": "SKU-001", "quantity": 3000, "required_date": "2026-06-01" }
{"can_meet": true, "earliest_completion": "2026-05-28", "capacity_ok": true}
📝
Step 3 — 接收 PO
ATP/CTP 確認無誤，接收客戶正式訂購單。
POST /api/v1/po
{ "po_id": "PO-20260518-001", "customer_id": "CUST-001", "order_date": "2026-05-18", "requested_delivery": "2026-06-01", "lines": [{"item_code": "SKU-001", "quantity": 3000, "unit_price": 150}] }
{"po_id": "PO-20260518-001", "status": "pending", "total_amount": 450000}
🔄
Step 4 — PO 轉 SO
PO 審核通過，PO-to-SO Agent 自動轉換為銷貨單（SO）。
POST /api/v1/po/PO-20260518-001/convert
{"so_id": "SO-20260518-001", "po_id": "PO-20260518-001", "status": "confirmed"}
🏭
Step 5 — 生產追蹤
SO 確認後進入生產，更新狀態追蹤進度。
PATCH /api/v1/so/SO-20260518-001
{ "status": "in_production" }
{"so_id": "SO-20260518-001", "status": "in_production", "updated_at": "..."}
📦
Step 6 — ASN 發出
生產完成，建立出貨通知（ASN）通知客戶準備收貨。
POST /api/v1/shipping/create
{ "so_id": "SO-20260518-001", "carrier": "DHL", "estimated_arrival": "2026-06-03" }
{"shipping_id": "SHP-xxx", "carrier": "DHL", "status": "pending"}
📦
Step 7 — Pick / Pack
倉庫揀貨包裝，更新出貨單狀態。
PATCH /api/v1/shipping/SHP-xxx/pack
{"shipping_id": "SHP-xxx", "status": "packed", "packed_at": "..."}
🚚
Step 8 — Ship 出貨
正式出貨，CS Agent 自動發送出貨通知給客戶。
PATCH /api/v1/shipping/SHP-xxx/ship
{"shipping_id": "SHP-xxx", "status": "shipped", "shipped_at": "..."}
📄
Step 9 — Invoice 開立發票
根據 SO 金額開立發票，CS Agent 通知客戶。
POST /api/v1/invoice/create
{"so_id": "SO-20260518-001", "total_amount": 450000}
{"invoice_id": "INV-xxx", "so_id": "SO-20260518-001", "total_amount": 450000}
🚢
Step 10 — Logistics 安排物流
安排報關和運輸，追蹤貨物位置。
POST /api/v1/logistics/arrange
{"shipping_id": "SHP-xxx", "warehouse": "TW-TPE", "destination": "US-LAX"}
{"tracking_no": "TRK-xxx", "status": "in_transit", "warehouse": "TW-TPE"}
✅
Step 11 — 確認到貨
貨物送達目的地，After Service Agent 啟動售後流程。
POST /api/v1/logistics/TRK-xxx/arrive
{"tracking_no": "TRK-xxx", "status": "delivered", "arrived_at": "2026-06-03"}
🎉

OTD 完整流程結束！

從詢單到到貨，ACME Corp 的 3,000 個 SKU-001 電子零件已完成全流程。

總計 11 個步驟，含 ATP/CTP 試算、PO→SO 轉換、生產追蹤、ASN、Pick/Pack/Ship、Invoice、Logistics。

5 OTD Agent 說明
Agent職責觸發條件
CS Agent 客服流程：詢單回覆、ASN 通知、出貨通知、Invoice 通知 新詢單 / 出貨完成
ATP/CTP Agent 交期試算：庫存可用性 + 產能可行性 收到詢單
PO-to-SO Agent PO 審核通過後轉換為 SO PO 審核完成
Logistics Agent 物流處理：報關、出貨、到貨確認 SO 建立 / 出貨完成
After Service Agent 售後服務：客戶反饋、案件追蹤 到貨完成 / 客戶回饋
