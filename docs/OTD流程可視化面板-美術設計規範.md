## 一、設計原則

1. **一目了然**：單屏即可判斷訂單整體進度
2. **即時感知**：異常狀態以色彩與動效優先傳達
3. **層級清晰**：流程圖（總覽）與明細表（細節）分層展示
4. **一致性**：同種狀態在任何視圖中視覺表現一致

## 三、OTD 流程圖設計

### 3.1 流程步驟（自上而下）

```
┌─────────────────────────────────────────────────────┐
│  訂單建立 (Order Created)  ──→  待處理                │
├─────────────────────────────────────────────────────┤
│  PO-to-SO Agent  ──→  將訂購單轉為出貨單             │
├─────────────────────────────────────────────────────┤
│  ATP/CTP Agent  ──→  可供貨量與交期確認              │
├─────────────────────────────────────────────────────┤
│  Customer Service Agent  ──→  客戶確認與通知         │
├─────────────────────────────────────────────────────┤
│  ERP 介接模擬層  ──→  與 ERP 系統對接                │
├─────────────────────────────────────────────────────┤
│  Logistics Agent  ──→  出貨安排與追蹤                │
└─────────────────────────────────────────────────────┘
```

### 3.2 流程圖視覺規範

- **流程卡（Step Card）**：寬度 100%，高度 64px
- **步驟間連接線**：2px solid `#334155`，箭頭 → 右側
- **當前步驟高亮**：邊框 2px solid `#3B82F6`，背景微光 `rgba(59,130,246,0.1)`
- **已完成步驟**：背景輕度 `rgba(34,197,94,0.08)`，連接線 `#22C55E`
- **異常步驟**：背景 `rgba(239,68,68,0.1)`，邊框 2px solid `#EF4444`，搭配閃爍動效
- **步驟圖標**：16px，使用 Lucide 圖標集

### 3.3 連接線樣式

```
───▶  正常流向
───────▶  已完成的流程（綠色）
───▶  異常跳過（紅色虛線）
───▶  回退重試（橙色）
```

## 六、動效規範

## 八、異常處理視覺

```
⚠️ 異常警示橫幅
┌────────────────────────────────────────────────┐
│  ⚠️  PO-2026-00123  ATP 超時 > 2h，請人工處理   │
│      [查看詳情]  [標記處理中]  [忽略]            │
└────────────────────────────────────────────────┘
```

異常級別：
- 🔴 **Critical**：流程中斷，需立即處理
- 🟡 **Warning**：超時風險，需關注
- 🔵 **Info**：待確認，提示性通知
